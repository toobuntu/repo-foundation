# Repo memory

> Durable per-repo knowledge (ADR 0022 in toobuntu/repo-foundation). Append dated entries (`## YYYY-MM-DD — Topic`); never rewrite history — a correction is a new entry naming what it supersedes. Decisions graduate to ADRs; org-wide knowledge belongs in `.ai/org/memory.md` (edited only in repo-foundation); volatile status belongs in the gitignored `.ai/progress.md`.
