# minimal-repo (a worked repo-foundation consumer)

A reduced example of what a `toobuntu/*` consumer looks like after `foundation-init` and the first sync. It is not a real project; it exists to show the two things a consumer most needs to understand — the managed region in a text file, and the deep-merge of `.claude/settings.json`.

## Files here

- `AGENTS.md` — a baseline-merge **text** target. The block between the `>>> repo-foundation managed baseline >>>` / `<<< end repo-foundation managed baseline <<<` sentinels is regenerated by every sync; everything outside it (the `## What this repo is` section) is the consumer's to edit. The sentinels are rendered in the target's own comment syntax — HTML comments here, `#` comments in a `.gitignore`.
- `.claude/settings.addenda.json` — a baseline-merge **JSON** target's input. `.claude/settings.json` has no comment syntax, so the sync cannot delimit a region in it. Instead the engine deep-merges repo-foundation's `provides/repo/settings.baseline.json` with this addenda file and regenerates `.claude/settings.json`. This repo commits only the addenda; the merged `settings.json` is generated.

## How the deep-merge behaves

Objects merge key by key; **arrays union** (so a consumer can only *add* to the org-wide permission and sandbox rails, never silently drop one); a scalar takes the consumer's value. Given the addenda here:

```jsonc
// .claude/settings.addenda.json
{
  "permissions": { "allow": ["Bash(./bin/minimal --version)"] },
  "sandbox": { "network": { "allowedDomains": ["example.test"] } }
}
```

the generated `.claude/settings.json` keeps the entire org-wide baseline and adds `Bash(./bin/minimal --version)` to `permissions.allow` and `example.test` to `sandbox.network.allowedDomains`. Re-running the sync is idempotent: the target is rebuilt from the baseline and the addenda each time, so a later change to either input propagates and nothing accumulates.

## Activating the hooks

A real consumer runs this once per clone (see the repository root README):

```sh
git config core.hooksPath .githooks
```
